Five stages. Zero trust required.

01
HARDWARE LAYER

Sovereign Node Mints Identity

On first boot, the device generates a BIP-39 12-word seed phrase from its hardware CSPRNG. PBKDF2-SHA512 stretches it into an Ed25519 keypair sealed inside the Android Keystore / iOS Secure Enclave. The Node ID is the SHA-256 fingerprint of the public key — unforgeable.

02
SENSOR LAYER

Live Sensor Telemetry Sampled

Every 30 seconds, the node samples ambient sensors — accelerometer, barometric pressure, ambient light. These values are aggregated and canonicalized into a deterministic sensor block that accompanies each heartbeat. A purely simulated node cannot reproduce real physical variance.

03
SIGNING LAYER

Hash-Chained Heartbeat Signed

The sensor block, sequence number, and previous chain-tip hash are assembled and signed with the node's Ed25519 private key. The SHA-256 of the signed payload becomes the new chain tip. Each beat is unforgeable and cryptographically linked to every beat before it.

04
ATTESTATION LAYER

Bureau Verifies & Attests

The Genesis Bureau independently verifies the node's signed Proof of Origin. It cross-references bureau-observed facts — first-seen timestamp, peak lifetime beats — against the node's claims. Sensor values are checked for physical plausibility. Flags are raised for any anomaly. Then the bureau signs an attestation bundle.

05
SCORE LAYER

Genesis Score Derived

The signed attestation bundle is mapped to a Genesis Score by any partner — no central authority owns the mapping. The score reflects node age, uptime consistency, sensor coherence, and chain integrity. A high score unlocks PREMIER tier earning rates.
